## Что сделано

### Часть 1 — Токенизация и статистика (30%)

| Шаг | Что | Результат |
|---|---|---|
| Токенизация двумя способами | `text.split()` vs `nltk.word_tokenize(language='russian')` | NLTK правильно отделяет запятые/тире/кавычки, `split()` склеивает их с словами (`лисицу,`, `«Каштанка»,`) |
| Нормализация | lowercase + удаление пунктуации (`string.punctuation` + русские кавычки/тире/`№`) | 5 844 токена |
| **N (всего токенов)** | | **5 844** |
| **V (размер словаря)** | | **2 377** |
| **V/N (type/token ratio)** | | **0.407** |
| Топ-20 частот | таблица + bar chart | в выводе ячейки |

Самые частые слова — служебные («и», «не», «в», «он», «она», «на», «по», «то») плюс имена («каштанка», «иван иваныч», «фёдор тимофеич»).

**Закон Ципфа.** На лог-лог графике частоты vs ранга получается прямая с наклоном ≈ −1: десяток служебных слов забирает значительную долю всех вхождений, длинный хвост — слова, встречающиеся один-два раза (hapax legomena). Это универсальное свойство естественного языка.

### Часть 2 — Редакционное расстояние (30%)

Реализована функция `edit_distance(s1, s2)` на динамическом программировании (numpy-матрица `(n+1)×(m+1)`). Каждая операция — вставка/удаление/замена — стоит 1. Возвращает и расстояние, и полную DP-матрицу.

**Тесты (10 пар):**

| s1 | s2 | distance |
|---|---|---|
| кот | кот | 0 |
| кот | код | 1 |
| мама | рама | 1 |
| intention | execution | 5 |
| доктор | тостер | 4 |
| привет | превед | 2 |
| kitten | sitting | 3 |
| Sunday | Saturday | 3 |
| алгоритм | логарифм | 4 |
| cat | cats | 1 |

Эталонные значения (включая `intention → execution = 5` из Jurafsky & Martin) совпадают.

**Поиск опечаток по словарю корпуса** — для каждой опечатки ищем 5 ближайших слов:

```
       чехав → [('чехова', 2), ('вела', 3), ('делал', 3), …]
     каштнка → [('каштанка', 1), ('каштанке', 2), …]
      хазяин → [('хозяин', 1), ('хозяина', 2), ('хозяев', 3), …]
       лошат → [('лошади', 2), ('лошадь', 2), …]
   спрашевал → [('спрашивая', 2), …]
```

На опечатках в одну букву метод даёт правильный ответ первой строкой. Слабые места: ограниченность словаря (нет «чехов» — только косвенные формы), равноправие всех замен (для русского полезно ещё учитывать клавиатурную близость и звуковое сходство).

### Часть 3 — N-граммовые языковые модели (40%)

- **Разбиение:** 80% / 20% по предложениям (`nltk.sent_tokenize` + `word_tokenize`). 394 предложения всего → 315 train / 79 test.
- **Биграммная модель:** 4 153 уникальных биграмм, словарь обучения V = 1 908. MLE-вероятности через `count(w1,w2) / count(w1)`.
- **Сглаживание Лапласа (add-1):** `P_lap(w_n|w_{n-1}) = (count(w_{n-1}, w_n) + 1) / (count(w_{n-1}) + V)`. Без него любая невиденная биграмма даёт `P = 0`, перплексия уходит в бесконечность.
- **Генерация (3 предложения, 10–15 слов):**
  1. *несомненный талант нужно было тихо тонким голоском ску-ску-ску… в спальню хозяина по опыту что же*
  2. *он умирает если бы ты поступил в одном из большой светлой комнате и оглядываясь каштанка*
  3. *вздохнул хозяин не было мало тётка закрыла глаза у него есть сильно дёргал её на*

  Биграммы даёт грамматически локально связные пары («он умирает», «тётка закрыла глаза»), но смысл рассыпается через несколько слов — модель не видит дальше одного слова назад.

- **Перплексия (Laplace) на тесте:**
  - Биграмма: **1 612.53**
  - Триграмма: **1 758.46**

  Триграмма проигрывает биграмме на малом корпусе из-за разреженности: число возможных триграмм ~V³, а в обучении видно лишь горстку. Add-1 с гигантским знаменателем `count(ctx) + V` размазывает массу слишком тонко. На больших корпусах картина обратная.

- **Перплексия vs размер train (10/25/50/80%):** монотонное снижение, выходящее на плато. Каждый дополнительный кусок данных приносит всё меньше — закон убывающей отдачи.

## Ключевые наблюдения

1. Type/Token ratio для художественного текста на русском в 6 тыс. слов — около 0.4. На большом корпусе будет меньше: добавляются повторяющиеся формы.
2. Закон Ципфа на лог-лог графике очень хорошо ложится на прямую `1/r` — даже на одной маленькой повести.
3. Edit-distance как метод исправления опечаток работает хорошо только при ограничении на расстояние ≤ 1–2 и при достаточно большом, чистом словаре.
4. Add-1 сглаживание — самый примитивный способ, и его «грубость» хорошо видна на сравнении биграмма/триграмма: чем больше V, тем больше штраф за невиденность.
